# product-configuration
Building a bike using principles from knowledge representation of configuration systems.

### Layout  
```
  ├── bike                   
  |    ├── components.lp        #Facts on components
  |    ├── instances.lp         #Facts on instances of each components
  |    ├── requires.lp          #Facts on requirements of instances/components
  |    ├── incompatibility.lp   #Facts on incompatibility between instances/components
  |    ├── user_req             #Files on user requirements             
  |    ├── show                 
  ├── configuration                   
  |    ├── encoding1.lp         #Encoding of product configuration
  ├── doc                   
  |    ├── probdef.txt           #Configuration problem definition with examples
  |    ├── pred.txt              #Definitions of the predicates used
  |    ├── iconfig.txt           #Interactive configuration ideas
  |    ├── modeling.md           #UML model of encoding 
  |    ├── bike_uml.png          #UML png file 
```
